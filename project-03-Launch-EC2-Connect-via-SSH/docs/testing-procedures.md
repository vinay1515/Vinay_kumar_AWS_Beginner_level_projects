# Testing Procedures

## Web Server Test
1. Copy the Public IPv4 address of the running EC2 instance.
2. Paste it into your browser (ensure it is using `http://`, not `https://`).
3. You should see the custom HTML page generated by the user data script.

## SSH Access (PuTTY)
1. Open PuTTY. Enter `ec2-user@<PUBLIC_IP>` as the host.
2. Under Connection > SSH > Auth > Credentials, select your `.ppk` file.
3. Open the connection and accept the host key.

## SSM Session Manager Access
1. In the EC2 Console, select the instance and click **Connect**.
2. Go to the **Session Manager** tab and click Connect. A browser-based terminal will open without requiring SSH keys.
